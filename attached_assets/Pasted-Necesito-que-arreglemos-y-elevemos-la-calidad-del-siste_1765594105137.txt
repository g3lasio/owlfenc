Necesito que arreglemos y elevemos la calidad del sistema multi-template. Ahorita el dropdown muestra tipos de documentos/contratos, pero quiero garantizar 3 cosas: (1) respaldo real por template, (2) calidad del contenido, (3) formato consistente con el contrato existente.

1) Cada opción del dropdown debe estar respaldada por un template real
	•	No quiero “items UI” que no tengan un template funcional detrás.
	•	Requiero que el dropdown se construya 100% desde TEMPLATE_REGISTRY:
	•	Si status = active → aparece con botón Generar
	•	Si status = coming_soon → aparece bloqueado, sin generar
	•	Si no existe template en registry → NO debe aparecer en UI

✅ Objetivo: cero opciones “fantasma”.

2) Calidad legal y completitud del contenido por documento
	•	Cada documento/contrato activo debe tener contenido completo, no placeholder.
	•	Cada template debe tener:
	•	Secciones claras (títulos, numeración, definiciones si aplica)
	•	Campos dinámicos bien ubicados
	•	Lenguaje coherente, consistente y profesional
	•	Validación de campos requeridos antes de generar PDF

Quiero una revisión de cada template activo y un checklist por template de:
	•	Campos requeridos
	•	Campos opcionales
	•	Secciones incluidas
	•	Riesgos (si falta algo importante)

3) Formato y estilo: usar como referencia el template del Independent Contractor Agreement

El contrato actual está bien presentado:
	•	Ordenado
	•	Paginado
	•	Con estilo consistente
	•	Footer “Powered by Mervin AI”

	•	Márgenes y saltos de página consistentes

	•	Header
